| md | 16px (`mb-4`, `gap-4`) | Gold-rule-to-eyebrow gap (`gap-4`), specialty card icon-to-heading (`gap-6` exception, see below) |
| lg | 24px (`mb-6`, `space-y-6`, `gap-6`, `pl-6`) | Eyebrow ‚Üí headline spacing (`mb-6`), paragraph stack (`space-y-6`), audience-item border-left padding (`pl-6`), specialty card inner flex gap (`gap-6`) |

| 2xl | 48px (`mb-12`, `gap-12 lg:gap-20`) | Bio grid gap on mobile (`gap-12`), specialty-block heading-to-list spacing (`mb-12`) |
| 3xl | 64px (`py-16`, `sm:py-20`) | ‚Äî (see `section-block` below for the larger spacing) |

This value is **encapsulated within `Button.astro`** ‚Äî MentorIntro passes `size="md"` to `<Button>` without overriding internal padding, so Button's interior padding is opaque to this component. MentorIntro does not own, declare, or expose this value. If the Button component ever changes its internal padding scale, that change is a Button concern, not a MentorIntro concern.

## Typography

Sizes and weights are inherited from `global.css @theme` tokens (lines 46-58). Declared token set is intentionally small ‚Äî Phase 6 uses 4 type roles, 2 weights + 1 approved exception.

| Role | Token | Size | Weight | Line Height | Usage in MentorIntro |
|------|-------|------|--------|-------------|----------------------|
| Body | `text-body` | `clamp(1rem, 1.25vw, 1.125rem)` ‚Üí 16‚Äì18px | 400 (regular) | 1.7 base / `leading-relaxed` (1.625) on paragraphs | Bio paragraphs, specialty card descriptions |
| Label | `text-xs` | 12px | 700 (bold) | 1 / `tracking-[0.2em]` | Gold eyebrow (`THE SPECIALIST`, `THE MENTOR`), button `size="sm"` |
| Heading H3 | `text-h3` | `clamp(1.5rem, 2vw + 1rem, 2rem)` ‚Üí 24‚Äì32px | 700 | 1.2 (base) | Specialty card title, audience heading |
| Heading H2 | `text-h2` | `clamp(2rem, 4vw + 0.25rem, 3.5rem)` ‚Üí 32‚Äì56px | 700 | 1.2 base / `leading-tight` (1.25) | Bio section headline (e.g. `Meet Lachlan`) |

**Weight inventory (declared contract): exactly 2 weights**
- `font-weight: 400` (regular) ‚Äî body copy, first-name italic emphasis
- `font-weight: 700` (bold) ‚Äî headings, labels, CTA button

**First-name italic emphasis:** rendered as `italic` (font-style) at **weight 400** ‚Äî the italic style alone provides sufficient visual differentiation against the surrounding H2 heading at weight 700. No third weight is introduced. Implementation: `<span class="text-white italic">{name}</span>` inside the H2 template.

**Approved exception ‚Äî `font-light` (300) for first-name emphasis:**

| Property | Value |
|----------|-------|
| Status | Approved exception ‚Äî NOT a third contract weight |
| Rationale | Must visually match Allan's existing H2 headline pattern in the same page |
| Locked by | Existing codebase: `src/pages/index.astro:121` uses `<span class="text-white italic font-light">Allan</span>` inside the About-section H2 |
| Scope | Single use: the first-name `<span>` inside the bio H2 only. Not used anywhere else in MentorIntro. |
| Review trigger | If Allan's section is ever refactored off `font-light`, MentorIntro must follow in the same commit |

The executor MUST render the first-name span as `<span class="text-white italic font-light">{name}</span>` to match the established site pattern at `index.astro:121`. This single-location decorative weight is carried by the approved exception above; it does not expand MentorIntro's 2-weight inventory.

Font family: `font-heading` (Playfair Display) for all H1/H2/H3 and the italic quote in bio; `font-body` (Inter) for everything else. Tokens already set on `<body>` and `h1-h6` base styles in `global.css:120-139`.

The table below declares the props' *expected copy shape* (what Phase 7 must supply) and the *fallback / default copy* for component states that have no corresponding prop (error / missing asset). This is the contract executor must honor in the component's prop-interface docs and alt-text defaults.

| Element | Copy Contract |
|---------|---------------|
| Primary CTA label (prop `cta.label`) | Verb + object, Title Case, 2‚Äì4 words. Phase 7 passes `"Book a Call"` or similar. Executor must **not** hardcode a fallback ‚Äî the label is a required prop; missing it is a TypeScript error. |
| Primary CTA target (prop `cta.href`) | Absolute URL. Phase 7 passes Allan's Calendly link as documented placeholder. Executor opens in new tab (`target="_blank" rel="noopener noreferrer"`) ‚Äî matches Allan's current CTA pattern. |

| Bio headline (prop `headline`) | Full sentence that **contains** `props.name` as a substring so the component can split on it for italic emphasis. Example: `"Meet Lachlan"` or `"Hey, I'm Lachlan"`. | Audience item title (prop `audienceItems[].title`) | Bold lead phrase, 2‚Äì4 words (e.g. `"Solo agency owners"`). Rendered as `<strong class="font-bold">`. |

## Interaction Contract

Phase 6's component carries three interaction surfaces. All are inherited from existing patterns; no new interaction JS is introduced.

| Surface | Behavior | Source Pattern |
|---------|----------|----------------|
| CTA button hover | Scale-up micro-translate (`hover:-translate-y-0.5`), white glow (`hover:shadow-[0_0_40px_rgba(255,255,255,0.3)]`), inner shimmer sweep (`btn-shimmer::after`) ‚Äî 500‚Äì700ms | `Button.astro:22-23` + `global.css:227-243` |
| Specialty card hover | Lift + scale (`translateY(-4px) scale(1.01)`), gold ring fade-in on border (`::before`), radial sheen tracks cursor (`::after` via JS-injected `--x`/`--y`) ‚Äî 600ms | `global.css:312-324` (`card-hover-glow`), JS wiring in `Base.astro:192-204` |
| Audience row hover | Border color shift `gold/30` ‚Üí `gold`, text color shift `white` ‚Üí `gold` ‚Äî 500ms | `index.astro:312-314` |
| Portrait image hover | Subtle zoom (`hover:scale-[1.02]`) ‚Äî 700ms duration on outer wrapper | `index.astro:139` |
| Scroll-triggered reveals | `data-motion` attribute contract ‚Äî no bespoke JS. Component tags elements with `blur-reveal`, `stagger-words`, `fade-up`, `skew-up`. Observer in `Base.astro:140-217` handles intersection + animation | `Base.astro:140-217` |
| Parallax on portrait | `data-parallax="-60"` on `<img>`; observer shifts transform on scroll. Specialty cards use `data-parallax="-20"` / `-40"` alternating | `Base.astro:140-217` + `index.astro:145,239,251` |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` in `global.css:175-185` forces `opacity: 1`, kills transform ‚Äî all `data-motion` elements visible without animation | `global.css:175-185` |

Phase 6 does NOT introduce:
- Any new hover utility classes
- Any new `@keyframes`
- Any component-scoped `<script>` block
- Any new entry in the Motion One observer's attribute vocabulary

If executor finds themselves writing a new animation keyframe or a new data-motion attribute type, stop ‚Äî that's scope creep.

## Image Pipeline Contract

Phase 6 emits exactly two WebP files. Visual characteristics are constrained by MENTOR-08 and the existing quality bar.

| File | Source | Target Width | Quality | File Size Target | Renders At |
|------|--------|--------------|---------|------------------|------------|
| `public/images/lachlan/lachlan-portrait.webp` | `_zip_temp/Lachlan/Portrait.JPG` (8.6 MB iPhone JPG) | 1600 px | 82 | **150‚Äì300 KB** (MENTOR-08) | ~700 px display width in `lg:col-span-7` ‚Äî 1600 px gives 2x retina headroom |
| `public/images/lachlan/lachlan-warroom.webp` | `_zip_temp/Lachlan/Picture_with_War_Room_members.JPG` (9.4 MB) | 1800 px | 80 | No hard budget (not displayed yet) | Placement deferred to Phase 7 or later ‚Äî Phase 6 only processes |

**Visual fidelity requirements** (inherited from MENTOR-08 + Pitfall 3 in RESEARCH.md):
1. EXIF orientation must be respected ‚Äî portrait renders upright, not sideways. Enforce via `sharp(...).rotate()` before `.resize()`.
2. Aspect ratio preserved ‚Äî sharp's `resize({ width, withoutEnlargement: true })` keeps source ratio; CSS container handles crop.
3. Color fidelity ‚Äî sharp defaults preserve the source color profile; no tone-mapping.
4. **Quality comparability to Allan's reference:** `public/images/heroes/alan-networking-2x.webp` = 177 KB at ~1600-1800 px / quality 80-82. Lachlan's portrait should land in the same visual neighborhood. If the photo still looks markedly "phone snapshot" next to Allan's editorial shot, executor should surface a release-note flag (Pitfall 3) rather than silently ship.
5. No `<Image>` component from `astro:assets` ‚Äî MENTOR-08 requires a stable, predictable output path (`public/images/lachlan/lachlan-portrait.webp`) so `MentorIntro.astro` can reference it as a plain URL string prop.

**Animation-relevant image property:** Because `data-parallax="-60"` shifts the image up to 60px vertically during scroll, the image wrapper's extended height (`h-[120%]`) and negative offset (`-top-[10%]`) must stay in place to prevent edge reveal. Changing these values on a per-phase basis would require re-testing ‚Äî for Phase 6, match Allan's values exactly.
