    **Replace with (single line added — `pipeline = pipeline.rotate();` right after `let pipeline = img;`):**
    ```javascript
    async function toWebp(src, dest, opts) {
      const img = sharp(src);
      const meta = await img.metadata();

      let pipeline = img;
      pipeline = pipeline.rotate();  // Auto-orient per EXIF; no-op when no orientation flag is present

      if (opts.width && meta.width > opts.width) {
        pipeline = pipeline.resize({ width: opts.width, withoutEnlargement: true });
      }
      if (opts.height && meta.height > opts.height) {
        pipeline = pipeline.resize({ height: opts.height, withoutEnlargement: true });
      }

      await pipeline.webp({ quality: opts.quality || 80 }).toFile(dest);
      return dest;
    }
    ```

    **Why this specific change:** sharp does NOT auto-apply EXIF orientation by default (see 06-RESEARCH.md Pitfall 1, citing sharp docs: "EXIF orientation is not taken into consideration"). `Portrait.JPG` is an 8.6 MB iPhone JPG that almost certainly has `Orientation=6` (rotate 90° CW) and will emit SIDEWAYS at width 1600 without `.rotate()`. Calling `.rotate()` with no argument applies EXIF-based auto-rotation AND strips the orientation tag — a no-op for images without orientation metadata, so existing logo/icon PNG outputs are unaffected.
