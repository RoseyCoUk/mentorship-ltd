    **Implementation notes (read before pasting):**

    1. **`headline.split(name)` safety:** If the consumer passes a `headline` that does NOT contain `name` as a substring, `split(name)` returns `[headline]` — the destructure `const [beforeName, afterName = ""]` uses a default of `""` to avoid `undefined`. The italic span will still render with `{name}` inside but visually adjacent to the full headline. This is acceptable fallback behavior. Do NOT add runtime validation — TypeScript + 06-UI-SPEC.md copy contract already require `headline` to contain `name`.

    2. **Flip implementation uses DOM ordering, NOT `flex-row-reverse`** (Pitfall 4). The grid is `grid-cols-12`; `flex-row-reverse` has no effect on grid containers. Two conditional blocks (`{flip && ...}` and `{!flip && ...}`) emit the image markup before or after the text column, which reorders the grid's DOM children and therefore their visual order.

    3. **Image-wrapper + parallax values match Allan's section exactly** (`aspect-[4/3] sm:aspect-[4/5] lg:aspect-[3/4]`, `h-[120%] -top-[10%]`, `data-parallax="-60"`). Do not alter these — Pitfall 6 (CLS) depends on the aspect-ratio parent reserving layout height before the image loads.

    4. **Specialty-card interior padding comes from `Card.astro` (`p-8 sm:p-10`)** — do NOT add a second `p-*` utility on the inner `<div>`. The `flex flex-col gap-6` inside is content layout, not padding.

    5. **The component MUST NOT be imported anywhere during Phase 6.** Phase 6 success criterion #4 ("renderable in isolation without breaking the page") depends on this. Phase 7 owns the import into `index.astro`. See Pitfall 5.

    6. **First-name italic span uses exactly `class="text-white italic font-light"`** — this matches Allan's pattern at `index.astro:121` and is the approved typography exception documented in 06-UI-SPEC.md ("Approved exception — `font-light` (300) for first-name emphasis"). Do not use `font-heading` / `font-normal` / any other weight.

    7. **CTA button props are locked:** `variant="primary"`, `size="lg"`, `target="_blank"`, `rel="noopener noreferrer"`. These match Allan's existing Calendly CTA pattern and the UI-SPEC "External link convention" row.

    8. **Audience-item em-dash:** Use a proper em-dash character ` — ` (U+2014 flanked by spaces). The `<strong>...</strong> — {desc}` markup matches `index.astro:312-314`.

    9. **Eyebrow label uppercase is done via CSS (`uppercase` utility), NOT via `.toUpperCase()` in the template** — per 06-UI-SPEC.md copy contract, "Executor must not apply `.toUpperCase()` in JS — `text-xs tracking-[0.2em] uppercase` handles it."
